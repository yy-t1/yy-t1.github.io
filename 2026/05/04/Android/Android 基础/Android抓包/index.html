Android抓包

抓包本质：观察APP与服务器之间传输的数据

抓包主要分为四大类

方法 原理 难度 HTTPS
代理抓包 中间人代理 ★ 可
VPN 抓包 本地 VPN 截获 ★★ 可
Root 抓包 底层 tcpdump ★★★ 原始流量
Hook 抓包 Hook API 拿明文 ★★★★ 最强

http：明文传输，抓到的包就是可直接读取的明文

https（HTTP + TLS）：抓到的是密文，无法直接读取。

https能防止抓包偷看通信内容，但是前提是没有主动信任攻击者的证书。

而https强度可以分为三种

未额外进行证书效验

APP → 系统证书库 → 验证服务器证书 → 建立 HTTPS

（所以只要证书是合法CA签发就行）

装证书就能抓到包

Android 7+之前

直接配置用户证书就能抓包

Android 7+之后

不再信任用户配置的证书。

存在两种思路：

1.将用户证书移到系统目录

2.配置系统强制信任用户证书

单向证书认证

客户端校验服务器证书–SSL Pinning

由上述可知，系统证书并非绝对可信，所以出现SSL。

SSL Pinning详解

将服务器证书“写死”在APP内，不再信任系统证书。

有两种方法

Certificate Pinning（证书锁定）

直接把服务器证书（server.crt)打包进APK，然后代码里只信任这个证书。

ps:但是证书会过期，如果APP不进行更新那么证书到期就会导致APP无法联网

以及对方服务器可能更新等等也会导致出现问题

Public Key Pinning（主流）

只绑定SPKI Hash，只要证书里的这个不变，那么APP就信任该服务器

（也就是将公钥指纹（SPKI Hash））写死进APP
